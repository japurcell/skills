Self-improve guidance (short)

- Keep `./AGENTS.md` minimal (project-wide, near-universal rules).
- Place scoped rules in `./<dir>/AGENTS.md` (module-specific test commands, validation, env quirks).
- Durable learnings: non-default commands, test/validation steps, repo-specific patterns, environment quirks, repeated human corrections.
- Always update destination in same change when moving guidance; avoid orphaned deletions.
