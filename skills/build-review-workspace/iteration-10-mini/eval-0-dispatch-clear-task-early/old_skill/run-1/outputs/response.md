Before the implementer runs, the manager should:
- Pick the next ready task (or a small already-ready independent wave).
- Confirm the task is clear enough to execute; if it’s ambiguous or conflicts with the plan, resolve that first.
- Dispatch immediately without drafting the solution or pre-reading large file sets.
- Keep the scope lean and pass only known context.

What belongs in the handoff:
- The task text and acceptance criteria.
- Known constraints.
- The validation commands already known for the repo.
- Only already-known file hints from the plan.

What must stay with the implementer:
- Repo discovery and pattern lookup.
- First-pass design.
- Reading the relevant code, tests, types, and commands.
- Writing the failing test.
- Implementing the minimum change.
- Running the relevant validation.
- Reporting back with results.
