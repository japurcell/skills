**Next step:** Do **not** update tracking. A reviewer result of `DONE_WITH_FINDINGS` means the work is still open. The manager should re-dispatch the subagent that owns the fix—typically the **implementer**—with the two correctness findings for those already-touched files. When that fix comes back `DONE`, send the affected scope back through **code-simplifier** and then the affected **code-reviewer** partition(s) again, reusing the same `review_scope_files` partitions unless the fix changed the surface enough to require new ones.

**Tracking can be updated only after** the re-fix/re-review loop finishes and **every reviewer returns `DONE`**. Only then should the manager update the plan and todo tracker, record the verification actually performed, and mark the task `done`.
