Phase 0 keeps repo-local overrides, but Phases 1–11 persist durable state in GitHub issues and comments instead of local per-work-item markdown files. For a new feature with no existing ISSUE, Phase 1 creates a lightweight parent issue labeled `agent:parent`, then a separate child issue labeled `agent:phase` and `phase:intake`. The intake child issue body is the structured intake artifact based on the intake template, it is attached to the parent with `gh api graphql ... addSubIssue`, and the parent remains the top-level tracker. Do not create `00-intake.md` or other local per-work-item markdown for Phase 1.
