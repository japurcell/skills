Manager checklist before dispatch
- Run addy-context-engineering and subagent-model-selection.
- Confirm the plan task is unambiguous and matches acceptance criteria; if ambiguous, resolve before dispatch.
- Do not explore the repo, draft a solution, or do design/coding.

What to include in the lean handoff
- Task text + success criteria (acceptance criteria).
- Known constraints and validation commands (exact commands to run).
- Only already-known file hints (paths named in the plan).
- Any repo rules or blocking requirements.

Work that stays with the implementer
- Repo discovery, pattern lookup, first-pass design.
- All code changes, tests, and verification (use the narrowest stack checks).
- Addressing routine questions about implementation details.
- Report DONE / DONE_WITH_CONCERNS per workflow.
