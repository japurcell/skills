**Do not dispatch an implementer yet.** In the build workflow, a task/plan mismatch is an explicit stop condition: after invoking `addy-context-engineering` and `subagent-model-selection`, if the next task **conflicts with the plan**, you **resolve that first** rather than dispatching. Here, “defer audit logging to phase 2” vs. “implement audit logging now” must be reconciled first; if the plan is wrong, that gets escalated to the human. Only once the scope is clear and consistent should an implementer be dispatched.
